# LabY

LabY is a GitHub organization for AI research, product experiments, and engineering collaboration.

## Working Norms

- Keep active work in repositories owned by `laby-ai`.
- Use issues for scoped tasks, decisions, and follow-up work.
- Use pull requests for code review and keep changes small enough to verify.
- Record verification steps in each pull request before merging.
- Prefer clear ownership: each repository should have a maintainer and a short README.

## Repository Defaults

New repositories should include:

- `README.md` with purpose, setup, and verification commands
- issue tracking enabled for task coordination
- pull request review before merging important changes
- branch protection once a repository becomes shared or production-facing

## Team Access

Use teams for access rather than adding collaborators one by one. A simple starting model is:

- owners: organization administration
- maintainers: repository maintenance and release decisions
- members: day-to-day contribution through issues and pull requests
